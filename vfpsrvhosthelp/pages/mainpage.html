VFPSrvHost

VFPSrvHost is a small executable that is able to run Visual FoxPro COM servers as a windows service.

The application is configured by a small XML file which contains all the information describing the windows services hosted by the executable. The format of the configuration file is documented in the Configuration page.

To implement a windows service in VFP at least two classes have to be implemented, a service "controller" class which will receive events when the service is requested to start, stop, pause or other service specific events. The other one is the "worker" class in which the actual work your service performs is implemented. The OS expects each service to handle control requests in a timely manner, that's why the "controller" and "worker" classes have to be decoupled. The controller will handle the events originating from the OS while the worker runs independently on it's own thread. How to implement a controller class is documented here

Installation, Configuration and Deinstallation of the windows service can be done by invoking the vfpsrvhost.exe on a DOS commandline which is described in the section Installation.
