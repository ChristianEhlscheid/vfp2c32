SetFileAttributesEx

Changes the attributes of a set of files/directories specified by a wildcard search.

SetFileAttributesEx(cFileSkeleton, ncAttributes[, nMaxRecursion]])

Parameters

cFileSkeleton

Fully qualified (drive:\path\wildcard) search string e.g. "C:\Winnt\*.dll"

ncAttributes (additional)
One or a combination of the following constants or a string where each character represents an attribute.
When passing a string a '-' sign before the attribute removes the attribute from the current files attributes and
a '+' sign adds the attribute.
When passing a numeric value, the file attributes are overwritten with the new value
AttributeCharacterDescription
FILE_ATTRIBUTE_ARCHIVEAA file or directory that is an archive file or directory. Applications typically use this attribute to mark files for backup or removal.
FILE_ATTRIBUTE_HIDDENHThe file or directory is hidden. It is not included in an ordinary directory listing.
FILE_ATTRIBUTE_NORMALNA file that does not have other attributes set. This attribute is valid only when used alone.
FILE_ATTRIBUTE_NOT_CONTENT_INDEXEDIThe file or directory is not to be indexed by the content indexing service.
FILE_ATTRIBUTE_OFFLINEOThe data of a file is not available immediately. This attribute indicates that the file data is physically moved to offline storage. This attribute is used by Remote Storage, which is the hierarchical storage management software. Applications should not arbitrarily change this attribute.
FILE_ATTRIBUTE_READONLYRA file that is read-only. Applications can read the file, but cannot write to it or delete it. This attribute is not honored on directories.
FILE_ATTRIBUTE_SYSTEMSA file or directory that the operating system uses a part of, or uses exclusively.
FILE_ATTRIBUTE_TEMPORARYTA file that is being used for temporary storage. File systems avoid writing data back to mass storage if sufficient cache memory is available, because typically, an application deletes a temporary file after the handle is closed. In that scenario, the system can entirely avoid writing the data. Otherwise, the data is written after the handle is closed.
nMaxRecursion (optional)

default = -1

-1 = no recursion
0 = recurse into all subdirectories
>0 = recurse into x subdirectories

Return Value

The number of files that were changed.
